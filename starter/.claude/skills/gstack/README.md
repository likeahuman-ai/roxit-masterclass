# gstack skills

A curated selection from [Garry Tan's gstack](https://github.com/garrytan/gstack) — YC CEO's Claude Code setup.
Use these slash commands at any point during the workshop.

| Command | When to use |
|---|---|
| `/office-hours` | Before starting — six forcing questions to scope your idea. Great for hackathon Day 1. |
| `/investigate` | Stuck on a bug? Systematic root-cause debugging — no fixes without a diagnosis. |
| `/review` | Before committing — staff engineer code review on your diff. |
| `/qa` | App built? QA test it, find bugs, fix them iteratively. |
| `/cso` | Security audit — OWASP Top 10 + dependency scan. |
| `/plan-eng-review` | Lock in your architecture before building. Eng manager questions. |
| `/design-review` | Designer's eye on your UI — spacing, hierarchy, AI slop patterns. |

These skills work alongside the role agents (`@product-owner`, `@backend-engineer`, `@frontend-engineer`).
A good hackathon flow: `/office-hours` → build with agents → `/review` → `/qa` → pitch.
